# 同步滚动实现说明

## 功能概述

实现了编辑器和预览窗口的双向同步滚动功能，当用户滚动其中一个面板时，另一个面板会自动同步滚动到相应位置。

## 实现原理

### 核心思路

使用**滚动比例**来计算目标滚动位置：

```
滚动比例 = scrollTop / (scrollHeight - clientHeight)
目标位置 = 滚动比例 × (目标容器scrollHeight - 目标容器clientHeight)
```

### 防止循环触发

使用 `syncLockRef` 防抖锁机制：
- 当一个面板滚动时，设置锁为 `true`
- 同步另一个面板的滚动
- 50ms 后解锁
- 锁定期间忽略滚动事件，避免循环触发

## 代码结构

### 1. Editor 组件改造

**文件**: `src/components/Editor/Editor.tsx`

使用 `forwardRef` 暴露滚动容器引用和方法：

```typescript
export interface EditorRef {
  getScrollContainer: () => HTMLElement | null;
  scrollTo: (top: number) => void;
}

export const Editor = forwardRef<EditorRef, EditorProps>((props, ref) => {
  const scrollContainerRef = useRef<HTMLElement | null>(null);
  
  // 暴露方法给父组件
  useImperativeHandle(ref, () => ({
    getScrollContainer: () => scrollContainerRef.current,
    scrollTo: (top: number) => {
      if (scrollContainerRef.current) {
        scrollContainerRef.current.scrollTop = top;
      }
    },
  }));
  
  // 获取 CodeMirror 的滚动容器 (.cm-scroller)
  scrollContainerRef.current = containerRef.current.querySelector('.cm-scroller');
  
  // 监听滚动事件
  if (scrollContainerRef.current && onScroll) {
    const handleScroll = () => {
      if (scrollContainerRef.current) {
        const { scrollTop, scrollHeight, clientHeight } = scrollContainerRef.current;
        onScroll(scrollTop, scrollHeight, clientHeight);
      }
    };
    scrollContainerRef.current.addEventListener('scroll', handleScroll);
  }
});
```

**关键点**：
- CodeMirror 的滚动容器是 `.cm-scroller`，不是外层容器
- 使用 `useImperativeHandle` 暴露方法
- 监听 `scroll` 事件并传递滚动信息

### 2. Preview 组件改造

**文件**: `src/components/Preview/Preview.tsx`

类似 Editor 的改造：

```typescript
export interface PreviewRef {
  getScrollContainer: () => HTMLElement | null;
  scrollTo: (top: number) => void;
}

export const Preview = forwardRef<PreviewRef, PreviewProps>((props, ref) => {
  const containerRef = useRef<HTMLDivElement>(null);
  
  useImperativeHandle(ref, () => ({
    getScrollContainer: () => containerRef.current,
    scrollTo: (top: number) => {
      if (containerRef.current) {
        containerRef.current.scrollTop = top;
      }
    },
  }));
  
  // 监听滚动事件
  useEffect(() => {
    if (!containerRef.current || !onScroll) return;
    
    const handleScroll = () => {
      if (containerRef.current) {
        const { scrollTop, scrollHeight, clientHeight } = containerRef.current;
        onScroll(scrollTop, scrollHeight, clientHeight);
      }
    };
    
    containerRef.current.addEventListener('scroll', handleScroll);
    return () => {
      containerRef.current?.removeEventListener('scroll', handleScroll);
    };
  }, [onScroll]);
});
```

**关键点**：
- Preview 的滚动容器就是 `.preview-container`
- 同样暴露 `getScrollContainer` 和 `scrollTo` 方法

### 3. App.tsx 中的同步逻辑

**文件**: `src/App.tsx`

实现双向同步滚动：

```typescript
function App() {
  const editorRef = useRef<EditorRef>(null);
  const previewScrollRef = useRef<PreviewRef>(null);
  const syncLockRef = useRef(false); // 防止循环同步
  
  // 编辑器 -> 预览
  const handleEditorScroll = useCallback((
    scrollTop: number, 
    scrollHeight: number, 
    clientHeight: number
  ) => {
    if (syncLockRef.current || !previewScrollRef.current) return;
    
    syncLockRef.current = true;
    const scrollRatio = scrollTop / (scrollHeight - clientHeight);
    const previewContainer = previewScrollRef.current.getScrollContainer();
    
    if (previewContainer) {
      const targetScrollTop = scrollRatio * (
        previewContainer.scrollHeight - previewContainer.clientHeight
      );
      previewScrollRef.current.scrollTo(targetScrollTop);
    }
    
    setTimeout(() => {
      syncLockRef.current = false;
    }, 50);
  }, []);
  
  // 预览 -> 编辑器
  const handlePreviewScroll = useCallback((
    scrollTop: number, 
    scrollHeight: number, 
    clientHeight: number
  ) => {
    if (syncLockRef.current || !editorRef.current) return;
    
    syncLockRef.current = true;
    const scrollRatio = scrollTop / (scrollHeight - clientHeight);
    const editorContainer = editorRef.current.getScrollContainer();
    
    if (editorContainer) {
      const targetScrollTop = scrollRatio * (
        editorContainer.scrollHeight - editorContainer.clientHeight
      );
      editorRef.current.scrollTo(targetScrollTop);
    }
    
    setTimeout(() => {
      syncLockRef.current = false;
    }, 50);
  }, []);
  
  return (
    <Editor
      ref={editorRef}
      onScroll={showEditor && showPreview ? handleEditorScroll : undefined}
      {...props}
    />
    <Preview
      ref={previewScrollRef}
      onScroll={showEditor && showPreview ? handlePreviewScroll : undefined}
      {...props}
    />
  );
}
```

**关键点**：
- 只有在双面板模式下才启用同步滚动
- 使用 `syncLockRef` 防止循环触发
- 50ms 延迟解锁，确保滚动事件处理完成

## 使用场景

### 启用条件

同步滚动只在以下情况下启用：
- ✅ 编辑器和预览都显示（双面板模式）
- ✅ 使用普通 Preview 组件（文档 < 50KB）

### 禁用情况

- ❌ 单面板模式（只显示编辑器或预览）
- ❌ 使用 VirtualPreview 组件（文档 > 50KB）

## 性能优化

### 1. 防抖锁机制

使用 `syncLockRef` 避免循环触发：
- 滚动 A → 同步 B → 不会触发 B → A 的同步

### 2. 条件监听

只在需要时添加滚动监听器：
```typescript
onScroll={showEditor && showPreview ? handleEditorScroll : undefined}
```

### 3. 大文档处理

超过 50KB 的文档使用 VirtualPreview，暂不支持同步滚动（避免性能问题）。

## 已知限制

1. **VirtualPreview 不支持同步滚动**
   - 原因：虚拟滚动的高度计算复杂，难以精确同步
   - 影响：超过 50KB 的大文档无法使用同步滚动

2. **滚动精度**
   - 使用比例计算，可能存在轻微偏差
   - 对于大多数场景已经足够精确

3. **Markdown 结构不对应**
   - 编辑器和预览的内容高度可能不完全对应
   - 使用比例而非行号映射，避免复杂的位置计算
