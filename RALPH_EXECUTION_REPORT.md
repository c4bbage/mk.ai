### Phase 1: Think & Plan 阶段

#### Orchestrator 决策

**需求理解**：
将 React + CodeMirror 6 + Tauri 的 Markdown 编辑器优化到 Typora 级流畅度（60fps、输入延迟 <16ms 感知、验收标准输入延迟 <50ms、启动 <2s、内存 <200MB）

**核心瓶颈总结**：
1. 编辑与预览强耦合 → 每次输入触发整文解析与整页重渲染
2. 重型模块在主线程同步执行（marked、KaTeX、Mermaid、代码高亮）
3. 预览为整页 DOM，虚拟滚动与高度管理不足
4. React 状态提升与引用不稳定导致级联重渲染
5. Worker 使用不彻底，缺少版本/取消与缓存策略
6. 冷启动加载过多资源

**优化策略**：
- 块级增量解析与渲染
- 解析迁移至 Web Worker
- 预览虚拟化与可视区懒渲染
- 输入到预览的自适应防抖/降优先级
- 重型依赖按需加载与代码分割
- 全面的 React 性能优化
- 全路径的性能监控与数据验证
